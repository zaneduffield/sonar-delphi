Why is this an issue?

Explicitly indexing a collection with a loop variable from 0 to Count - 1 is unnecessarily verbose and error-prone when the collection provides an enumerator.

Noncompliant code example 
  var
    I: Integer;
    L: TList;
  begin
    for I := 0 to L.Count - 1 do begin
      Foo(L[I]);
    end;
  end.
Compliant code example 
  var
    E: Pointer;
    L: TList;
  begin
    for E in L do begin
      Foo(E);
    end;
  end.

How to fix it

  1. Convert the loop statement to the 'for-in' syntax
  2. Update all uses of the original loop variable as an index to the new loop variable
  3. Clean up any now-unused variables and consider removing any redundant variables

Exceptions

No issue is raised if the index variable is used in the loop body for something other than indexing the enumerable collection.
